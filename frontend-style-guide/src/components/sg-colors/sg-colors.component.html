Colour System

A unified and strictly defined colour system ensures visual consistency across the application.

Usage Rules

  • Primary colours: branding and structural UI elements
  • Secondary colours: backgrounds and layout surfaces
  • Accent colours: interactive elements only (buttons, links, states)
  • Maximum of 2-3 accent colours per view
  • Standardised colour codes must always be used
  • Contrast ratios must always meet accessibility requirements

Contrast Mode / Dark Mode

  • The default product interface uses the light theme documented in this style guide
  • Dark mode must use --color-primary-dark as the main background and white or sand-light text on top
  • Interactive states keep the same accent colours so success, warning, error and information remain recognisable
  • Surfaces in dark mode must use a visible border with --color-primary-light or --color-muted-rose
  • Dark mode is optional, but if implemented it must meet WCAG 2.1 AA contrast and reuse the same component structure
