Accessibility

Standard & Compliance

  • WCAG 2.1 Level AA compliance required
  • Accessibility applies to visual design, component behaviour, wording and technical implementation

Visual Accessibility

  • Contrast ratios:
    • Text: 4.5:1
    • Headings: 3:1
  • No reliance on colour alone
  • Text must use the documented type scale and readable line height
  • Small text must still meet contrast requirements and remain legible on mobile viewports

Interaction Accessibility

  • Keyboard navigation required
  • Screen reader support required
  • Visible focus indicators required
  • Click targets >= 2.75rem
  • Proper form labels and validation required
  • Interactive states must be available for hover, focus, active, disabled and loading where relevant

Semantic HTML (Best Practices)

  • Use semantic HTML elements (<main>, <section>, <nav>) instead of generic <div> elements where possible
  • Use IDs and ARIA labels where needed

ARIA Roles & Labels

  • Use ARIA attributes to describe element roles and states
  • Use ARIA labels to clarify function (e.g. button purpose)
  • Use ARIA attributes when semantic HTML is insufficient
<button aria-label="Save content">Save</button>

Keyboard Navigation

  • Make sure all interactive elements are accessible via keyboard
  • Focus indicators must clearly highlight active elements
  • Full keyboard operability required

Images & Alternative Text

  • All images must include meaningful alt text
