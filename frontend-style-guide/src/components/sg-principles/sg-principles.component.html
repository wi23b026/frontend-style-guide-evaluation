Overall Design Principles

  • Consistency across all components
  • Usability and accessibility as core principles
  • Functional design over decoration
  • Scalable and maintainable system

Usability Principles

Usability Heuristics

  • Visibility of system status: The system should always tell users what is happening and provide clear feedback after actions.
  • Match between system and real life: The interface should use words and ideas that users already know from real life.
  • User control and freedom: Users should be able to undo actions or leave processes easily.
  • Consistency and standards: Similar things should look and work the same way everywhere.
  • Error prevention: The system should try to stop mistakes before they happen.
  • Recognition over recall: Users should not have to remember too much information; important options should be visible.
  • Flexibility and efficiency: The system should be easy for beginners and fast for experienced users, including shortcut support where useful.
  • Aesthetic and minimalist design: The design should be clean and show only necessary information.
  • Error recovery: If an error happens, the system should clearly explain it and help users fix the problem.
  • Help and documentation: Help information should be available when needed, simple to understand and easy to find.

Practical Usability Implementation

  • System feedback: interactive elements must provide visible default, hover, focus, active, disabled and loading states where relevant
  • Error handling: validation rules must use clear error messages, visible error states and safe recovery paths
  • Status indicators: use consistent icons, labels and colours to communicate success, warning, error and information states
  • Safe interaction patterns: destructive actions must be visually distinct, while cancel and close actions stay easy to find without danger styling
  • Reduced cognitive load: follow the wording rules, clear navigation structures and minimal layouts
  • Illustrative examples: each component should include usage, example and characteristics to support fast practical application

Illustration & Visual Language

  • Minimal, soft, organic visual style
  • Used sparingly
  • Never distract from content

Icons

  • Outline style only
  • Consistent usage
  • No mixed icon libraries

Images

  • Natural lighting
  • Authentic style
  • No heavy filters or stock imagery
